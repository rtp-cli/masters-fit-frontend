# File Naming

- Use kebab-case for file names (e.g., `use-onboarding-controller.ts`)
- Use PascalCase for component names (e.g., `UseOnboardingController`)

# Function Naming

- Use camelCase for function names (e.g., `startWorkoutGeneration`)

# Variable Naming

- Use camelCase for variable names (e.g., `isGeneratingWorkout`)

# For components have use in their name, use the prefix "use" (e.g., `useOnboardingController`)

- name there function/class accordingly (e.g., `useOnboardingController`)

# Class Naming

- Use PascalCase for class names (e.g., `OnboardingController`)

# Constant Naming

- Use UPPER_SNAKE_CASE for constant names (e.g., `REGENERATION_TYPE_WEEKLY`)

# Enum Naming

- Use PascalCase for enum names (e.g., `RegenerationType`)

# Coding Conventions

- Write code in components ui + business logic
- in app/ directory only render components or navigation based upon cases
